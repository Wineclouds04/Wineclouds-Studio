# WinUI 3 桌面应用壳设计

**日期：** 2026-07-14  
**状态：** 待开发者审查  
**阶段：** 第一阶段（仅 UI 应用壳）

## 1. 目标与范围

开发一个仅支持 Windows 的 C#/.NET 桌面应用，采用 WinUI 3。主窗口的左侧为深色分组导航栏，右侧为与导航项对应的内容区域。

第一阶段只交付界面骨架：导航可切换到对应的占位页面；不实现真实业务服务、数据访问、登录、网络通信、权限、插件系统或模块隔离。

## 2. 已确认的设计决策

| 项目 | 决策 |
| --- | --- |
| 平台 | 仅 Windows |
| UI 框架 | WinUI 3（Windows App SDK） |
| 项目形态 | 单一应用项目，不拆分独立模块类库 |
| 导航布局 | 左侧深色分组侧栏；右侧内容区 |
| 导航控件 | `NavigationView` + 内容 `Frame` |
| 首阶段页面 | 首页，以及模块 A–F 的占位页 |
| 测试与审查 | 由开发者负责，不属于本设计交付范围 |

## 3. 界面设计

### 3.1 窗口骨架

主窗口由一个浅蓝色左侧导航栏和一个白色右侧内容区组成：

```text
┌──────────────────────┬─────────────────────────────────────────────┐
│ Wineclouds Studio    │ 当前页面标题                                │
│                      │                                             │
│ 工作台               │  ┌───────────────────────────────────────┐  │
│  · 首页              │  │                                       │  │
│                      │  │        当前模块占位内容               │  │
│ 服务模块             │  │      “功能建设中”                     │  │
│  · 模块 A–F          │  │                                       │  │
│                      │  └───────────────────────────────────────┘  │
└──────────────────────┴─────────────────────────────────────────────┘
```

### 3.2 左侧导航

- 使用浅蓝色背景，与右侧白色内容区形成柔和层级。
- 顶部显示应用名称 `Wineclouds Studio`。
- 使用分组标题组织导航项：第一阶段预置“工作台”和“服务模块”。
- 导航项同时显示图标和中文文字；选中项使用浅蓝背景和深蓝前景色。
- 第一阶段不提供收起、拖动排序、固定、最近访问或动态增减导航项。

### 3.3 右侧内容区

- 由 `Frame` 承载页面切换。
- 每个占位页显示页面标题、模块名称和“功能建设中”提示。
- 页面只负责视觉占位，不调用任何业务服务。
- 首次启动默认显示“首页”。

## 4. 架构与目录

第一阶段采用单一 WinUI 3 应用项目，并通过文件夹维持足够清晰的职责边界：

```text
WinecloudsStudio/
├── Shell/          # 主窗口、导航定义与应用壳
├── Pages/          # 首页和各模块占位页面
├── ViewModels/     # 预留给未来页面状态；本阶段不创建
├── Services/       # 预留给未来应用级服务；本阶段不创建
├── Assets/         # 图标、图片及主题资源
└── App.xaml        # 应用资源与启动配置
```

不引入 `Contracts`、独立模块项目、反射扫描、动态加载或跨进程通信。后续只有当多个模块的业务、发布节奏或团队分工确实需要独立演进时，才评估拆分。

## 5. 导航与数据流

```text
用户点击 NavigationView 项
        ↓
主窗口处理选择变化
        ↓
根据导航项映射目标 Page 类型
        ↓
Frame 显示该 Page
        ↓
Page 显示对应模块的静态占位内容
```

导航映射应集中在主窗口代码中维护，避免每个占位页反向依赖主窗口。第一阶段使用静态映射；不需要数据库、远程配置、依赖注入或 MVVM 基础设施。

## 6. 状态与异常处理

- 仅维护当前选中的导航项和当前显示页面。
- 如导航项未配置目标页面或导航失败，右侧显示统一的“页面暂不可用”占位状态，而不是让窗口退出。
- 页面加载期间不显示业务加载动画，因为第一阶段不访问任何外部数据。
- 未实现的功能一律以占位内容表达，不创建伪造数据或空服务调用。

## 7. 非功能约束

- 界面应适配常见 Windows 缩放比例，保证导航文字和右侧标题在高 DPI 下清晰可读。
- 所有首阶段文本使用中文。
- 键盘焦点应可见，导航项可被键盘选择；颜色对比度应保持可读性。
- 应用启动后直接打开首页，不要求登录或首次配置。

## 8. 交付与验收边界

本设计要求的第一阶段交付物：

- 一个可启动的 WinUI 3 应用。
- 深色分组左侧导航栏。
- 包含首页，以及模块 A–F 的可切换占位页。
- 每个页面在右侧内容区显示其名称和“功能建设中”状态。

测试、代码审查、质量门禁和发布验证由开发者负责。本设计不规定具体测试框架、测试用例或审查流程。

## 9. 后续扩展（不纳入第一阶段）

- 接入真实业务服务与数据源。
- 引入设置、用户权限、主题切换和本地化资源。
- 依据实际复杂度，将稳定且边界清晰的业务区域抽取为独立项目。
- 需要时再设计模块注册、插件化或进程隔离机制。
